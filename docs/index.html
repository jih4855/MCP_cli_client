MCP CLI Client

CLI 환경에서 MCP(Model Context Protocol) server를 활용할 수 있는 다중 LLM 지원 client

빠른 시작 GitHub
Terminal
$ mcp-client
MCP CLI Client 시작
user: 현재 시간과 시스템 정보 알려줘
도구 호출: get_current_time
도구 호출: get_ubuntu_system_info

프로젝트 목적

Linux/Unix 환경에서 MCP server와 연동하여 다양한 도구를 활용할 수 있는 CLI 기반 AI assistant를 제공합니다.

저장소 클론

프로젝트를 로컬에 복사합니다.

Terminal
$ git clone <repository-url>
$ cd typer_cli

가상환경 설정 및 설치

가상환경을 생성하고 패키지를 글로벌 명령어로 설치합니다.

Terminal
$ python -m venv venv
$ source venv/bin/activate
$ pip install -r requirements.txt
$ pip install -e .

어디서든 실행

이제 어디서든 mcp-client 명령어를 사용할 수 있습니다.

Terminal
$ mcp-client
가상환경 활성화 없이 실행 가능

저장소 클론 및 가상환경

개발용으로 프로젝트를 설정합니다.

Terminal
$ git clone <repository-url>
$ cd typer_cli
$ python -m venv venv
$ source venv/bin/activate

의존성 설치

필요한 패키지들을 설치합니다.

Terminal
$ pip install -r requirements.txt

개발 모드 실행

가상환경을 활성화한 후 실행합니다.

Terminal
$ python main.py
가상환경 활성화 필요

pipx 설치

pipx를 먼저 설치합니다.

Terminal
$ pip install pipx

프로젝트 설치

격리된 환경에 프로젝트를 설치합니다.

Terminal
$ pipx install .

격리된 환경에서 실행

pipx로 설치된 명령어를 실행합니다.

Terminal
$ mcp-client
격리된 환경에서 실행

설정

환경변수 (.env)

Gemini API 키를 설정합니다.

.env
api_key=your_gemini_api_key_here

LLM 설정 (config.yaml)

LLM 모델과 시스템 프롬프트를 설정합니다.

config.yaml
system_prompt: |
당신은 도움이 되는 AI assistant입니다.
model: gemini-2.5-flash
provider: gemini
api_base: None
max_context: 20
session_id: "default"

MCP 서버 설정 (mcp_config.json)

사용할 MCP 서버들을 설정합니다.

mcp_config.json
{
"mcpServers": {
"ubuntu-info": {
"command": "python",
"args": ["mcp_server/MCP_official.py"]
},
"filesystem": {
"command": "npx",
"args": ["-y", "@modelcontextprotocol/server-filesystem@latest", "/home"]
}
}
}

📖 사용법

기본 대화

간단한 대화를 나눠보세요. "끝"이라고 입력하면 종료됩니다.

Terminal
user: 안녕하세요!
assistant: 안녕하세요! 무엇을 도와드릴까요?
user: 끝
대화 종료

도구 사용

MCP 서버의 도구들을 자동으로 병렬 실행합니다.

user: 현재 시간과 시스템 정보를 알려주세요
도구 호출: get_current_time
도구 호출: get_ubuntu_system_info
